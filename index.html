PROJECTS

AMSAT is a Twitter bot that I built in JavaScript, the summer before starting university. The Twitter bot in and of itself is fairly boilerplate; it uses RESTful and streaming Twitter API calls to monitor users, and tweet when it wants to. The more impressive feature of the bot is that it uses a Markov generator to construct tweets, which I built from scratch in JavaScript. This was the main challenge of the project, but also the most fun to make! Markov chains can be used in natural language processing, and are what power the predictive text on your phone. I use them to generate tweets from sample text, which I then push to the bot's Twitter account. The whole thing is hosted on Heroku, and runs more or less constantly. It doesn't make much sense (Markov chains aren't really known for their accuracy, as we all know from autocorrect), but it produces some pretty amusing stuff!

laz0rb0i is an integrated system that relies on user input and a series of motors and mirrors to create a laser light display on a flat surface. It was built using an Omega2, a laser pointer, several motors and mirrors, a motor controller, some home-grown C++ code to run the whole thing, and lots of love. Several challenges were encountered while building laz0rb0i, most of which related to actually controlling the motors using GPIO pins from the Omega2. These were overcome by creating a command-line tool which accepted user input, then translated it into a series of more basic commands the Omega2 could work with. Because the Omega2 is a computer in its own right, the project is run by SSHing into the Omega2 terminal.

ASCIIfy is a Chrome extension which transforms all the images on a webpage into ASCII pictures. It's written in JavaScript, and built to use Chrome's extension format. It waits until the user clicks the actionable extension icon, then transforms all the images it can into ASCII art. It does so by reading in pixel data from the image, breaking it up into rectangles of consistent size, then finding the average grayscale value of each square. It then assigns each square an ASCII-encoded character based on its greyscale value. I encountered many challenges while working on this particular project. Working with images is difficult due to cross-origin security protocols, and in order to read the pixel data I needed to find a way around it. Luckily, I was able to find a solution by creating a new image with the same image data, and modifying its 'origin' property manually.
