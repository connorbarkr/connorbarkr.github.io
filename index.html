PROJECTS

AMSAT is a Twitter bot that I built for fun, the summer before starting university. The Twitter bot in and of itself is fairly boilerplate; it uses RESTful and streaming Twitter API calls to monitor users, and tweet when it wants to. The more impressive feature of the bot is that it uses a Markov generator to construct tweets, which I built from scratch in JavaScript. This was the main challenge of the project, but also the most fun to make! Markov chains can be used in natural language processing, and are what power the predictive text on your phone. I use them to generate tweets from sample text, which I then push to the bot's Twitter account. The whole thing is hosted on Heroku, and runs more or less constantly. It doesn't make much sense (Markov chains aren't really known for their accuracy, as we all know from autocorrect), but it produces some pretty amusing stuff!

laz0rb0i is an integrated system that relies on user input and a series of motors and mirrors to create a laser light display on a flat surface. It was built using an Omega2, a laser pointer, several motors and mirrors, a motor controller, some home-grown code to run the whole thing, and lots of love. Several challenges were encountered while building laz0rb0i, most of which related to actually controlling the motors using GPIO pins from the Omega2. These were overcome by creating a command-line tool which accepted user input, then translated it into a series of more basic commands the Omega2 could work with. Because the Omega2 is a computer in its own right, the project is run by SSHing into the Omega2 terminal.

This is totally not a remake of Space Invaders for fun. Just kidding! I used pygame to build a remake of this classic arcade game, with custom art to put it all together. Some of the challenges I faced were learning about control schemes; I ended up using "flags" to indicate whether or not an action was currently occurring. I also had to make creative use of the "random" function to generate a new background of stars each game. Saving high scores was also a challenge, although I overcame it using a text file, stored in the same location as the game's code. This was necessary because there's no application that contains the game; it's run using pygame, from the console.

PORTAL

january - april 2018 | developer

I'm currently working on the Portal team, who make the student life application and website for the Univerity of Waterloo. This is our home-grown one stop shop for student needs, ranging from class and exam times and locations to news feeds to checking the balance on your student meal card. On the team, I worked on a variety of tasks, including creating the analytics tracker for the app and site from scratch. To do this, I designed the frontend using AngularJS 1.6 and Bootstrap. The backend was written in C#, where I wrote several API calls which pull from our database to process information and pass it to the service. The biggest challenges here were learning how to process high volumes of data both correctly and efficiently, and working with clients to better understand their requests, needs, and expectations for the final project.
